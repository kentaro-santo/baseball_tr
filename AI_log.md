# 野球部 トレーニング・数値管理アプリ 開発ログ

## プロジェクト概要
部活動での体重やトレーニングの数値管理を目的とした野球専用のアプリケーション。選手のモチベーションアップ、怪我の予防、効果的なトレーニング計画の策定を目指す。

## 開発環境
- フォルダ: `C:\Users\shant\OneDrive\ドキュメント\baseball_tr`
- 技術スタック: HTML, CSS (Vanilla), JavaScript, Chart.js

## 実装済みの機能 (2026-03-13 時点)
初期の静的なHTMLモックアップ（UIのみ）を作成。
以下のタブと機能のUIを実装済み。

1. **ダッシュボード**: 現在の体重、球速、ウエイトMAXなどの重要指標の概要と、日々の推移を視覚的に確認できるグラフ（体重、球速など）。
2. **体重・体組成**: 日付ごとの体重、体脂肪率、コンディションを記録する入力フォーム。
3. **ウエイトトレーニング**: ベンチプレスやスクワットなどの種目ごとに、重量と回数・セット数を記録するフォーム。
4. **野球指標**: 球速、スイングスピード、50m走タイム、遠投などの野球に特化した数値を記録するフォーム。
5. **目標設定**: 各選手の目標（例：球速145km/h、体重80kgなど）に対する現在の達成率をプログレスバーで視覚化し、進捗管理を行うUI。
6. **ランキング**: 球速やウエイトのMAX重量などでチームメイトとランキング形式で比較できる画面。

### デザイン
- プレミアム・ダークモードを採用。
- グラスモーフィズム（ガラスのような透け感）を取り入れた洗練されたUI。
- モチベーションを維持・向上させるためのダイナミックで直感的なデザイン。

## 開発ログ

### 2026-03-10: データ保存処理の実装 (LocalStorage)
- `index.html` の各入力フォーム（体重・体組成、ウエイトトレーニング、野球指標）にIDを付与。
- `app.js` を更新し、フォーム送信時にブラウザの LocalStorage にデータを記録（`weightRecords`, `trainingRecords`, `statsRecords`）する処理を実装。

### 2026-03-11 ~ 03-12: モバイルUIの刷新と選手管理機能の実装
- **レスポンシブ対応の強化**: モバイル端末での操作性を向上させるため、スライディングサイドバーとオーバーレイを実装。
- **表示モード切替機能**: PC版とスマホ版の表示をワンタップで切り替えられるボタントグルを導入。
- **選手管理画面の追加**: 選手の登録・編集・削除ができるインターフェースを実装。マスター権限と選手権限による表示制御（ロール管理）の基礎を構築。
- **UI/UXの洗練**: グラスモーフィズムデザインの採用と、Chart.jsによるデータの視覚化（体重・球速グラフ）の改善。

### 2026-03-15: コメント機能とヘッダー認証UIの追加
- **マスターからの個別コメント送信**: マスター（監督・コーチ）が複数人いることを想定し、ログイン時に「送信者名」を入力するフィールドを追加。選手管理タブから各選手へ固有のフィードバックコメントを送る機能を実装した。
- **選手ダッシュボードでの新着コメント表示**: 選手本人がログインした際、ダッシュボードの最上部にマスターから届いた最新のメッセージと送信者名を表示する機能を実装した。
- **ヘッダーログイン/ログアウトUI**: 画面右上のヘッダー部分に、PC/スマホ問わず常にアクセスしやすい「ログイン」「ログアウト」ボタンを配置。未ログイン時はログインモーダルをワンクリックで起動し、ログイン中は即座にログアウトできるように改善した。
- **バグ修正**: 「選手管理」タブのメッセージ送信ボタン（手紙アイコン）が反応しない不具合（関数のスコープ・消滅エラー）を改修し、正常にモーダルが開くよう修正。

### 2026-03-14: マスター権限の強化とデータ出力機能の実装
- **マスター・ログイン機能**: URLパラメータ（`?role=master`）による簡易アクセスを廃止し、専用のパスワード（`admin2026`）を用いたマスター用ログイン画面を実装。
- **データ一括出力（CSV Export）**: マスター権限でログインした場合のみ、全選手の体重・トレーニング・野球指標データを統合したCSVファイルを出力・ダウンロードできる機能を追加。
- **記録の所有者変更**: マスター権限に限り、既存の記録データ（体重・トレーニング・指標）の編集画面から、その記録を「誰の記録にするか（所有者）」をドロップダウンで変更できる機能を追加。誤って別の選手として登録されたデータを修正可能にした。
- **モバイルUIの修正**: スマホ版デプロイ時に、サイドバーのドロワーを開いた際にメニューの文字が消えてしまうCSSの不具合（メディアクエリの優先順位エラー）を修正。

### 2026-03-17: チャット機能の双方向化・専用タブ実装とリアルタイム通知
- **専用チャットタブの追加**: サイドメニューに「チャット」項目を新設。モーダルだけでなく、全画面でLINEのような感覚でやり取りできる専用画面を実装した。
- **双方向チャットシステム**: 選手とマスターがリアルタイムで会話可能。マスター側は選手選択、選手側はマスターとの直接対話ができる。
- **通知システムの高度化**: ベルアイコンからのタブ切り替え連動。未読がある場合、ダッシュボードにも強調表示される仕組みを導入。
- **既読・未読管理の徹底**: メッセージ閲覧時の自動既読処理と、双方への既読ステータス表示。

### 2026-03-24: アカウント機能の充実 (プロフィール・設定モーダルの実装)
- **プロフィール設定**: 自身の基本情報（名前、学年、ポジション、背番号）や「ひとこと目標」を自己更新できる機能を追加。
- **アバターカスタマイズ**: ランダム生成だったアイコンを、好みのスタイルと文字列（シード）で自由にカスタマイズ可能にした。
- **テーマカラー**: アプリ全体のアクセントカラーを5色（ブルー、オレンジ、グリーン、レッド、パープル）から個別に選べる機能を実装。
- **アカウント管理**: Firebase Authを利用した「パスワードの変更」および「アカウントの完全削除（Danger Zone）」機能を実装。
